# 美术资源制作清单

给实际画图用的详细规格表。怎么把画好的图放进项目、怎么触发自动替换，见 [ART.md](ART.md)——那份是"how to"，这份是"具体每张图要画多大、放哪、什么格式"的清单。

## 通用规格（对所有图都适用）

| 项目 | 要求 |
|---|---|
| 格式 | **PNG**（需要透明通道就用 PNG-24/32；不透明的可以用 PNG-24） |
| 背景 | 除了"每关背景大图"这一类，其余所有图（角色/怪物/道具/砖块等）都要**透明背景**，不要留白色或彩色背景板，否则会变成一个方块贴在角色周围 |
| 色彩模式 | RGB / RGBA，8-bit 每通道即可，不需要额外的色彩管理（sRGB 默认即可） |
| 分辨率 | 不需要考虑"DPI"或视网膜倍图（@2x/@3x）——下表给的数字就是最终显示的物理像素尺寸，画多大最后就显示多大（会做拉伸/缩放匹配，但按下表尺寸画拉伸最小、最清晰） |
| 尺寸容差 | 不需要精确到个位数像素，但**宽高比**尽量接近下表数字，比例差太多会导致明显拉伸变形 |
| 风格方向 | Q 版萌向：大头小身、圆润轮廓、明快高饱和度配色，参考 [PLAN.md](PLAN.md) 第 2 节美术风格定位 |
| 命名 | 文件名必须和下表"文件名"列完全一致（大小写、下划线都要对，不能加多余后缀） |

## 七、每关背景大图（5 张，可选，一次性整体换皮）

如果想直接换掉现在的天空蓝底+程序化云朵观感，画一张贯穿整关的背景大图。**这一类背景图不需要透明背景**，可以是完整的不透明画面（因为背景本来就在最底层）。

| 文件名 | 存放路径 | 尺寸（px，宽 × 高，已按关卡实际长度精确计算） | 关卡名 |
|---|---|---|---|
| `bg-1-1.png` | `public/assets/backgrounds/` | **10368 × 960** | 阳光草地 |
| `bg-1-2.png` | `public/assets/backgrounds/` | **13440 × 1152** | 蘑菇森林 |
| `bg-1-3.png` | `public/assets/backgrounds/` | **14400 × 1536** | 云端漫步 |
| `bg-1-4.png` | `public/assets/backgrounds/` | **16800 × 1536** | 沙丘迷城 |
| `bg-1-5.png` | `public/assets/backgrounds/` | **17760 × 1344** | 糖果城堡 |

- 这几张图非常宽（横向卷轴关卡的全长），建议先画一个较窄的"可重复/可延展"的构图打底，再用软件里的画布拉伸/平铺工具铺到目标尺寸，不需要真的从头到尾手绘一张上万像素宽的图。
- 只画其中几关也可以，没画背景大图的关卡会继续显示现在的天空+云朵效果，互不影响。
- 这张图只影响天空/远景，**地面本身（`ground_tile.png`）、角色、怪物、道具不受影响**，会正常叠加显示在背景大图上面。

## 汇总：需要建的文件夹

```
public/assets/sprites/       ← 除背景大图外的所有图都放这里
public/assets/backgrounds/   ← 只有"每关背景大图"放这里
```

两个文件夹如果不存在，创建后直接把文件丢进去即可，不需要额外配置。

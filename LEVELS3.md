# 第 11–15 关（第三世界 3-1 ~ 3-5）关卡设计与实施计划

前两个世界已完成并实测收敛（1-x 见 [LEVELS.md](LEVELS.md)，2-x 见 [LEVELS2.md](LEVELS2.md)）。**第三世界（本篇）也已全部建关+实测收敛**——本篇是第三阶段的**玩法设计 + 工程实施计划**，对应 [PLAN.md](PLAN.md) 的 **Phase 13：第三世界内容制作（已完成）**；子阶段 13.0~13.6 的实测结果、已知局限见 PLAN.md Phase 13 小节与各关"验收清单"。

> 世界主题定位：第一世界教"基础动作"（踩踏/踢壳/平台/机关/管道），第二世界教"环境力"（冰/暗/带/风）。第三世界的机制轴是**互动机关与节奏**——弹簧、状态切换方块、强制卷轴、上涨岩浆追逐，最后由一场"机关同步型"Boss 战收拢。玩家的操作词汇不再变多，变多的是"关卡自己会动、会变"的规则。

## 0. 设计基准（含前两阶段全部实测教训，硬约束）

坐标一律以"格"为单位（1 格 = 96px = 32px 设计值 × WORLD_SCALE 3）。以下每一条都在前两个世界里真实咬过人，第三世界无条件沿用：

| 约束 | 数值/规则 | 出处 |
|---|---|---|
| 标准跳跃 | 最高 ~3.5 格、水平 ~5 格（助跑 6 格）；台阶高差 ≤3 格 | 第一阶段 |
| 起跳走廊净空 | 起跳点上方 **~5.5 行内不得有任何实心平台**（跳跃冲量固定必撞头）；折返阶梯错列或同列间隔 ≥6 行 | 2-4 实测 |
| 世界顶边界 | y≤3 行起跳会被顶边截断滞空（4 格间隙变不可跳）；高处间隙按截断后 ~3 格设计 | 2-4 实测 |
| 复活点 | 脚部不得嵌入地面、不得与管道等实心机关水平重叠（含 P2 的 -1.5 格偏移落点）；`npm run audit` 规则 4.5 自动拦截 | 2-1/2-4 实测 |
| 移动平台可追性 | 玩家需要追赶/登乘的平台，速度必须明显低于该环境下的持续移动速度（水下 450、陆地 600/900）；水下桥 ≤~85 设计值 | 1-5 水桥回归 |
| 登乘判定 | 粘附嵌入容差 48px（覆盖 60Hz 两帧坠速 + 单向平台跳穿上桥）；**登乘类交互必须用"随机相位真实起跳"测试** | 1-3/1-5 实测 |
| 引擎位移 | 手动搬运实体只改 GameObject，**严禁再调 `body.updateFromGameObject()`**（postUpdate 反馈回路：位移翻倍+吞玩家速度） | 传送带/浮桥实测 |
| 组默认值 | 无重力/特殊物理实体入 physics group 后必须重申设置（`noGravity` 标志模式） | 蝙蝠实测 |
| 确认键 | 所有结算/菜单画面的确认一律"先松开再按下"轮询（防按住穿透连锁） | 选关/结算实测 |
| 净空/缺口 | 砖块与悬空条下方 ≥2 格；有效缺口 ≥2 格；黑暗区缺口边缘 1 格内必须有发光金币标记 | 第一/二阶段 |
| 金币可拾取 | 金币所在列的"地面-天花板"口袋 ≥ 小形态 120px 且在跳跃可达带内（水下按 2× 跳）| 1-4 口袋金币 |
| 封闭 Boss 房 | 必须有"墙外复活"逃生阀（场内无人→拆墙清 Boss 回待触发态） | 2-5 软锁 |

**每关必跑**：`npm run audit`（几何+复活点+金币）→ 机制专项 Playwright（真实按键/随机相位）→ 低配 bot 通关 → 十五关全量回归。

## 3-1｜弹簧牧场（Springfield Meadows）

**定位**：第三世界开场，零压力教会"弹簧改变跳跃预算"。弹簧不是障碍是玩具——先给爽感，再用它够到常规跳够不到的地方。

**新机制规格——弹簧垫（springs）**：
- JSON：`"springs": [{"x", "y", "size"}]`，size = `"small" | "big"`
- 物理：站上/落上弹簧顶面 → 向上冲量。小弹簧弹高 **5 格**（v=1860，普通跳 3.5 格），大弹簧弹高 **8 格**（v=2350，< 最大坠速 2700 不会穿地）。按住跳跃键落在弹簧上额外 +15% 弹高（奖励时机，非必需操作）
- 实现：静态实体（1 格宽、半格高视觉，物理体只有顶面判定条）；玩家 feet 落入顶面判定 → `setVelocityY(-v)` + 压缩回弹动画（缩放视觉贴图，不缩物理体——setScale 物理坑）+ 弹簧音效。敌人/道具不触发（保持可预测）
- **弹高与顶边界**：大弹簧 8 格弹高 + 玩家身高 ≈ 10 行，放置位置上方必须留 ≥10 行净空（设计基准第 2 条），建关时逐个核对
- 视觉：红白螺旋方块（占位：红色矩形+白色横条），压缩时高度减半

## 3-2｜红蓝回廊（Crimson & Azure Halls）

**定位**：解谜型平台关。全关铺设红/蓝两色方块，任意时刻只有一色是实心的；踩击切换开关全场翻转。路线设计成"必须在正确的颜色状态下通过每一段"，节奏感来自"边跑边规划下一次翻转"。

**新机制规格——红蓝切换方块（switchBlocks / colorSwitches）**：
- JSON：`"switchBlocks": [{"x","y","widthTiles","color"}]`（color=`"red"|"blue"`），`"colorSwitches": [{"x","y"}]`
- 全局状态 `scene.activeColor`（初始 red）：激活色方块实心+满饱和；非激活色 body.enable=false + 半透明轮廓（**必须保留轮廓**——玩家要能预判翻转后的地形，全隐藏是坑）
- 切换开关：地面按钮（顶一下/踩一下都触发），全场翻转 + 音效 + 屏幕轻微闪白 0.1s；开关冷却 0.5s 防连发
- **防压死规则**：翻转瞬间若有玩家与即将实心化的方块重叠 → 该方块保持虚化直到玩家离开（逐块延迟实心化，不是取消翻转）。这是本关最大的工程细节，必须专项测试

## 3-3｜疾风飞车（Gale Rail Canyon）

**定位**：全游戏第一条**强制卷轴**关。镜头以固定速度向右推进，被左边缘追上即损命。玩家从"探索节奏"切换到"被节奏推着走"，所有旧机制（平台、弹簧、管道夹）在时间压力下重新变难。

**新机制规格——强制卷轴（autoScroll）**：
- JSON：关卡根字段 `"autoScroll": {"speedPx": 80, "startTile": 8, "endTile": 236}`（设计值 px/s，×WORLD_SCALE = 240px/s ≈ 40% 走速——可以从容小跑，但不能停下来观光）

**测试策略**（延续被验证有效的做法，新增三条本阶段特有）：
- 弹簧/卷轴/岩浆全部属于"改变玩家纵向预算/时间预算"的机制——专项测试必须包含**最坏时机**用例（弹簧起跳瞬间被卷轴追上、翻转瞬间站在方块里、岩浆 pause 结束瞬间起跳）
- 切换方块的**单色路线完整性**进审计工具（任一颜色状态下主线可达），防止建关时铺出"两色都断路"的死局
- Boss 相位与开关的时序用"状态机穷举"脚本验证（幻影态踩开关/实体态踩开关/冷却中踩开关/双人同时踩两个开关 四象限）

**风险预判**：
1. **防压死延迟实心化**（3-2）是全新交互，排 13.2 早期做 spike；退路是"翻转时把重叠玩家轻推出方块"（次优但安全）
2. **强制卷轴×双人**组合面广（镜头接管/软墙/泡泡跟随/左缘判死），13.3 给足 3 天；单人先行验收，双人作为独立出口条件
3. **岩浆×检查点联动**若出现边界 bug（复活即死/岩浆倒流），退路是"死亡一律回卷轴前安全层"（牺牲一点节奏保正确性）
4. 三个"时间压力"机制（卷轴/岩浆/幻影循环）都依赖 `scene.time`——暂停/失焦时 Phaser 时钟行为要统一核查一次（失焦回来岩浆瞬移是硬伤）
